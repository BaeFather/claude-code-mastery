# 개발자 포트폴리오 사이트 - ROADMAP

## 📋 프로젝트 개요

개발자 이력서 및 포트폴리오를 소개하는 **반응형 웹 포트폴리오 사이트**를 구축합니다.

### 기술 스택
- **마크업**: HTML5
- **스타일링**: CSS3, TailwindCSS
- **인터랙션**: JavaScript (ES6+)
- **배포**: GitHub Pages / Vercel / Netlify (미결정)

### 핵심 목표
- 전문적이고 현대적인 개인 포트폴리오 웹사이트 구현
- 반응형 디자인으로 모든 디바이스 지원
- 부드러운 사용자 경험과 인터랙티브 요소 제공

---

## 🚀 개발 로드맵

### Phase 0: 프로젝트 셋업
**상태**: 🔜 대기 중  
**예상 소요 시간**: 1-2시간

- [ ] 프로젝트 디렉토리 구조 설계
- [ ] npm 프로젝트 초기화 (`package.json` 생성)
- [ ] TailwindCSS 설치 및 설정
- [ ] 개발 서버 설정 (라이브 리로드)
- [ ] `.gitignore`, `README.md` 작성
- [ ] 기본 HTML 템플릿 작성

**산출물**: 프로젝트 기본 구조, 빌드 및 개발 환경 준비

---

### Phase 1: HTML 구조 작성
**상태**: 🔜 대기 중  
**예상 소요 시간**: 3-4시간

HTML5 시맨틱 마크업을 사용하여 포트폴리오 사이트의 뼈대를 작성합니다.

#### 페이지 섹션
- [ ] **Navigation Bar**: 로고, 메뉴 링크 (Hero, About, Skills, Experience, Projects, Contact)
- [ ] **Hero Section**: 자기소개, 프로필 사진, CTA 버튼
- [ ] **About Section**: 경력 소개, 교육 배경, 주요 성취
- [ ] **Skills Section**: 기술 스택 (분류별: Languages, Frontend, Backend, Tools)
- [ ] **Experience Section**: 직무 경력, 회사, 프로젝트
- [ ] **Projects Section**: 주요 포트폴리오 프로젝트 (카드 레이아웃)
- [ ] **Contact Section**: 연락처, 소셜 미디어 링크, 이메일 폼
- [ ] **Footer**: 저작권, 추가 링크

**산출물**: 완성된 HTML 구조 (`index.html`)

---

### Phase 2: 스타일링 (TailwindCSS)
**상태**: 🔜 대기 중  
**예상 소요 시간**: 6-8시간

TailwindCSS를 활용하여 전문적이고 모던한 스타일을 적용합니다.

#### 디자인 요소
- [ ] **컬러 팔레트 정의**: Primary, Secondary, Neutral 색상
- [ ] **Typography**: 폰트 패밀리, 크기, 가중치 설정
- [ ] **반응형 레이아웃**: 모바일(< 640px), 태블릿(640px-1024px), 데스크톱(> 1024px)
- [ ] **각 섹션 스타일링**: 배경, 텍스트, 간격, 카드
- [ ] **버튼, 폼 요소**: 기본 스타일, 호버 효과
- [ ] **다크모드 지원** (선택): `dark:` 접두사로 다크 테마
- [ ] **애니메이션 준비**: Tailwind Animation, Custom CSS 클래스

**산출물**: 반응형 웹 디자인, 모든 섹션의 시각적 완성

---

### Phase 3: 인터랙티브 기능
**상태**: 🔜 대기 중  
**예상 소요 시간**: 4-5시간

JavaScript를 사용하여 사용자 상호작용을 구현합니다.

#### 기능 요소
- [ ] **부드러운 스크롤**: 네비게이션 링크 클릭 시 부드러운 이동
- [ ] **모바일 네비게이션**: 햄버거 메뉴, 반응형 드로워
- [ ] **스크롤 애니메이션**: 요소가 뷰포트에 진입할 때 페이드인/슬라이드인
- [ ] **Active 네비게이션**: 현재 섹션에 따른 활성화 표시
- [ ] **Dark Mode 토글** (선택): 사용자가 테마 전환
- [ ] **프로젝트 카드 상호작용**: 호버 효과, 링크 오버레이
- [ ] **이메일 폼 유효성 검사**: 클라이언트 사이드 검증

**산출물**: 완전하게 인터랙티브한 포트폴리오 사이트

---

### Phase 4: 콘텐츠 채우기
**상태**: 🔜 대기 중  
**예상 소요 시간**: 2-3시간

실제 개인 정보와 프로젝트 내용을 입력합니다.

- [ ] 프로필 사진 추가
- [ ] About 섹션에 경력 내용 작성
- [ ] Skills 데이터 입력
- [ ] Experience 섹션 채우기 (직책, 회사, 기간, 설명)
- [ ] Projects 섹션 작성 (프로젝트 제목, 설명, 기술 스택, 링크, 스크린샷/이미지)
- [ ] Contact 정보 입력 (이메일, 소셜 미디어)

**산출물**: 실제 데이터가 반영된 완성 포트폴리오

---

### Phase 5: 최적화 & 테스트
**상태**: 🔜 대기 중  
**예상 소요 시간**: 3-4시간

성능, 접근성, 브라우저 호환성을 검증합니다.

#### 성능 최적화
- [ ] 이미지 최적화 (WebP, 해상도별 크기 조정)
- [ ] CSS 퍼징 (TailwindCSS 빌드 최적화)
- [ ] 불필요한 JavaScript 제거
- [ ] Lighthouse 점수 80점 이상 목표

#### 접근성 (Accessibility)
- [ ] ARIA 레이블 추가
- [ ] 키보드 네비게이션 지원
- [ ] 색상 대비 확인
- [ ] 스크린 리더 테스트

#### 브라우저 호환성
- [ ] Chrome, Firefox, Safari, Edge 테스트
- [ ] 모바일 브라우저 (iOS Safari, Chrome Mobile) 테스트
- [ ] 폰트 로딩 폴백 확인

#### SEO 최적화
- [ ] Meta 태그 작성 (description, keywords, og:*)
- [ ] Structured Data (Schema.org) 추가
- [ ] 사이트맵 생성

**산출물**: 최적화된 프로덕션 준비 완료

---

### Phase 6: 배포
**상태**: 🔜 대기 중  
**예상 소요 시간**: 1-2시간

선택된 플랫폼으로 라이브 배포합니다.

#### 배포 옵션 (아직 미결정)
1. **GitHub Pages**: 무료, Git 연동, 간단함
2. **Vercel**: 자동 빌드, CDN, 우수한 성능
3. **Netlify**: 자동 빌드, 폼 제출 지원, DNS 관리

#### 배포 단계
- [ ] 배포 플랫폼 선택 및 계정 생성
- [ ] 저장소 연결 및 빌드 설정
- [ ] 커스텀 도메인 설정 (선택)
- [ ] 자동 배포 파이프라인 설정
- [ ] 라이브 사이트 최종 검증

**산출물**: 공개 가능한 라이브 포트폴리오 사이트
